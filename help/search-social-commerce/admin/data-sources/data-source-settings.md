# [!DNL Google Analytics] impostazioni origine dati

| Sezione | Parametro | Descrizione |
| ---- | ---- | ---- |
| [!UICONTROL Connect to Google Analytics] | [!UICONTROL Google Analytics Account ID] | ID del [!DNL Google Analytics] account da cui vengono estratti i dati. Tutto l’utilizzo dell’API per richiamare i dati viene fatturato all’account specificato. L’account deve assegnare le autorizzazioni &quot;Read and Analyze&quot; (Lettura e analisi) all’indirizzo e-mail specificato.<br><br>Per trovare il tuo ID, accedi a [!DNL Google Analytics]. In alto a sinistra, fai clic su **[!DNL All accounts]** per aprire un elenco degli account. L’ID di ciascun account si trova sotto il nome dell’account. |
|  | [!UICONTROL Google Analytics Login] | Specificare l&#39;indirizzo di accesso/posta elettronica da utilizzare per accedere ai dati per questa origine dati. L’accesso deve essere registrato in un [!DNL Google] e disporre delle autorizzazioni di &quot;lettura e analisi&quot; per [!DNL Google Analytics] account. Consulta la [istruzioni per l’assegnazione delle autorizzazioni utente in [!DNL Google Analytics]](https://support.google.com/analytics/answer/9305587).<br><br><b>Nota:</b> Se si modifica la password per questo account di accesso, tutte le connessioni aperte all&#39;account verranno chiuse. Per riprendere la sincronizzazione dei dati, torna a questa pagina e [riautentica](data-source-reauthenticate.md). |
| [!UICONTROL Account Details] | [!UICONTROL Google Analytics Account Name] | (Sola lettura; solo account connessi) Il nome dell&#39;account. |
|  | [!UICONTROL Google Analytics Property] | La proprietà (sito web, app mobile o dispositivo) da cui raccogliere i dati per il [!DNL Google Analytics] account.<br><br> Per integrare le metriche per più proprietà, imposta un’origine dati separata per ciascuna proprietà. |
|  | [!UICONTROL Google Analytics View] | Visualizzazione contenente il set di dati a cui si desidera accedere. Se la proprietà dispone di più visualizzazioni, è consigliabile richiamare la visualizzazione non filtrata al livello più alto, che contiene tutti i dati.<br><br>Per integrare le metriche per più viste per la stessa proprietà, imposta un’origine dati separata per ciascuna vista. Assicurati che le visualizzazioni non includano dati sovrapposti. |
|  | [!UICONTROL Google Analytics Dimension] | Il [!DNL Google Analytics] dimensione personalizzata compilata con il valore del parametro della stringa di query &quot;ef_id&quot; Adobe Advertising. Se nell’elenco non viene visualizzata la dimensione corretta, contatta il [!DNL Google Analytics] team di implementazione.<br><br>&quot;ef_id&quot; viene utilizzato come chiave primaria da cui trasmettere i dati [!DNL Google Analytics] per Adobi di pubblicità. |
| [!UICONTROL Import Metrics] | [!UICONTROL Available Metrics] | Tutte le metriche disponibili per il valore specificato [!DNL Google Analytics] proprietà e visualizzazione non importate per l&#39;origine dati, organizzate per categoria. L’elenco include il nome descrittivo e il nome back-end importati (che iniziano con &quot;ga.&quot;) per ogni metrica. Il [!UICONTROL Refresh] aggiorna l’elenco con tutte le nuove metriche presenti in [!DNL Google Analytics].<br><br>Per importare una metrica disponibile, trascinala nella [!UICONTROL Selected Metrics] sezione.<br><br>Consulta &quot;[Disponibile [!DNL Google Analytics] Metriche in Advertising Cloud](data-source-ga-metrics.md).&quot; |
|  | Metriche selezionate | Tutte le metriche per il valore specificato [!DNL Google Analytics] proprietà e vista importate per l&#39;origine dati e stato di convalida per ciascuna metrica. L’elenco include il nome descrittivo e il nome back-end importati (che iniziano con &quot;`ga.`&quot;) per ogni metrica. Ogni origine dati può includere quattro metriche di traffico predefinite che non è possibile rimuovere ([!UICONTROL Pageviews], [!UICONTROL Sessions], [!UICONTROL Bounces], e [!UICONTROL Session Duration]) e fino a 16 metriche o metriche valide aggiuntive senza dati. Puoi modificare l’elenco delle metriche in qualsiasi momento.<br><br>Per importare una metrica, selezionala nella [!UICONTROL Available Metrics] e trascinarlo qui.<br><br><b>Attenzione:</b> [!DNL Google Analytics] consente fino a 10 metriche in un singolo feed di dati. Ogni origine dati in Search, Social e Commerce può includere fino a due feed con un totale di 20 metriche, ma l’utilizzo di un secondo feed raddoppia le chiamate API a [!DNL Google Analytics]. Se disponi di molte metriche, seleziona solo quelle che desideri utilizzare negli obiettivi per l’ottimizzazione. È possibile visualizzare le quote per questo progetto in [il [!DNL Google API Console]](https://console.developers.google.com/apis/api/analytics-json.googleapis.com/quotas). Ulteriori informazioni su [quote e limiti di chiamata per le richieste API a [!DNL Google Analytics]](https://developers.google.com/analytics/devguides/reporting/core/v4/limits-quotas).<br><br><b>Note:</b><br><ul><li>Il nome descrittivo viene importato come nome visualizzato per una metrica nella [!UICONTROL Admin] > [!UICONTROL Transactions Properties] e viene aggiunto con &quot;_ga:&lt;metric_tag configured=&quot;&quot; in=&quot;&quot; the=&quot;&quot; metric=&quot;&quot; tag=&quot;&quot; field=&quot;&quot;></code>&quot; (come &quot;Pageviews_ga:UK&quot;). Facoltativamente, puoi modificare il nome visualizzato degli obiettivi e delle metriche personalizzate, ma i nomi visualizzati per tutte le metriche comuni vengono sovrascritti ogni giorno con i nomi descrittivi in [!DNL Google Analytics], aggiunto con i tag specificati.</li><li>Visualizzazioni di pagina, sessioni, frequenza di rimbalzo (calcolata come rimbalzi/sessioni) e durata della sessione vengono automaticamente considerati negli algoritmi di offerta del portfolio. Puoi aggiungere manualmente qualsiasi altra metrica a una finalità di portfolio.</li><li>Quando rimuovi una metrica dall’origine dati, Adobe Advertising mantiene i dati storici secondo la normale [criteri di conservazione dei dati](/help/search-social-commerce/reports/data-used-for-reports.md).</li></ul> |
| Tag della metrica | Tag della metrica | Un tag da aggiungere a ogni selezionato [!DNL Google Analytics] metrica in Adobe Advertising, preceduta da &quot;_ga:</code>&quot; (come &quot;Pageviews_ga:&lt;metric_tag>&quot;). Il tag può contenere da 2 a 5 caratteri alfanumerici e deve essere univoco per l’inserzionista.<br><br>Questo tag consente di identificare l’origine dati per ogni metrica. Questo tag è particolarmente importante quando imposti più origini dati perché ogni origine dati include alcuni degli stessi nomi di metriche (tra cui [!UICONTROL Pageviews], [!UICONTROL Sessions], [!UICONTROL Bounces], e [!UICONTROL Session Duration]e potenzialmente altre metriche). L’aggiunta di tag di metrica diversi impedisce la duplicazione dei nomi delle metriche.<br><br>Ad esempio, se imposti integrazioni separate per la proprietà UK e la proprietà JP, puoi utilizzare &quot;UK&quot; e &quot;JP&quot; come tag metrici. Il [!UICONTROL Pageviews] Le metriche per le due proprietà vengono quindi visualizzate di conseguenza in Adobe Advertising come &quot;Pageviews_ga:UK&quot; e &quot;Pageviews_ga:JP&quot;. |
