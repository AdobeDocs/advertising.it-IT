# Informazioni su [!UICONTROL Advertising Insights]

[!UICONTROL Advertising Insights] presentare dati visivi e actionable sui portfolio ottimizzati e attivi che contengono campagne attive.

Ogni informazione viene generata su richiesta e l’output è un file che puoi scaricare o un rapporto su schermo. Puoi eliminare le istanze generate dei file di approfondimento.

## Tipi di [!UICONTROL Advertising Insights]

| Insight | Descrizione |
| --- | --- |
| [!UICONTROL AMO-AA Tracking Discrepancy] | (Solo per inserzionisti con un’integrazione Advertising-Adobe Analytics di Adobi) Identifica i problemi di tracciamento, caratterizzati da giornate con un rapporto di clic sulle istanze al di fuori della normale varianza o con una varianza del 20% o superiore. La relazione include:<ul><li>A [!DNL Microsoft® PowerPoint] un file con un riepilogo della varianza dei dati; una tabella e grafici di tendenza che mostrano la varianza per campagna per le campagne con il maggior numero di problemi di tracciamento; suggerimenti per la risoluzione dei problemi; e una spiegazione della metodologia del rapporto.</li><li>A [!DNL Microsoft® Excel] file con dati di riepilogo per ogni campagna nel portfolio e dati giornalieri del mese precedente per ogni campagna.</li></ul> |
| [!UICONTROL Attribution Analysis] | ([!DNL PowerPoint] (formato di presentazione) Indica quando un modello di attribuzione diverso può migliorare i modelli di ricavo e l’ottimizzazione per un singolo portfolio. |
| [!UICONTROL Campaign Caps] | ([!DNL PowerPoint] formato di presentazione) Indica se la spesa di un singolo portfolio negli ultimi 30 giorni è stata limitata dai limiti di budget della campagna e consiglia di modificare le impostazioni del portfolio per ottenere un ritorno ottimale sull&#39;investimento. |
| [!UICONTROL Day of Week] | ([!DNL PowerPoint] formato di presentazione) Indica le prestazioni di un singolo portfolio per giorno della settimana (DOW) negli ultimi 30 giorni e consiglia gli obiettivi di spesa DOW per aumentare il ritorno sull&#39;investimento.<br><br>**NOTA:** I Portfoli senza spesa sufficiente o che non sono stati in grado di spendere per il targeting negli ultimi due giorni non sono disponibili per l’approfondimento. |
| [!UICONTROL Delayed Revenue] | Misura il ritardo di conversione (il tempo trascorso tra un clic di annuncio e una successiva conversione) di un portfolio e mostra eventuali differenze in [ricavi ponderati](/help/search-social-commerce/glossary.md#w-x), ROI e precisione del modello a causa del ritardo. Questa analisi ti consente di rilevare le modifiche recenti nelle metriche delle prestazioni attribuite alla data di clic e di visualizzare l’impatto del reporting per data di clic rispetto alla data della transazione.<br><br>La descrizione include [!DNL Excel] cartella di lavoro con singoli fogli per ricavi ponderati, ricavi non ponderati, precisione dei ricavi e fattori di ritardo. Include inoltre un [!DNL Powerpoint] file con vari grafici. |
| [!UICONTROL Event Path] | Identifica il modo in cui i diversi canali e campagne guidano le conversioni analizzando i percorsi degli eventi e le conversioni multi-touch. Puoi caricare un file in formato XLSX e ZIP (XLSX compresso) e a) estrarre gli eventi o b) analizzare gli eventi classificati. |
| [!UICONTROL Google Account Audit] | (Per [!DNL Google Ads] account (molto utile per la fase di pre-vendita) Fornisce una panoramica delle prestazioni di un account ed evidenzia le carenze per mostrare come viene gestito in modo efficace. L’audit include informazioni sulle offerte, i tipi di corrispondenza, i limiti di budget, le prestazioni del giorno della settimana, le prestazioni mobili e altro ancora. Per generare le informazioni, devi caricare 1) [!DNL Google Ads] report di campagne, parole chiave e cronologia delle modifiche per l’account da [!DNL Google Ads] interfaccia utente web e b) un file bulksheet per l’account da [!DNL Google Ads Editor] applicazione. Tutti i file devono essere in formato CSV, TSV, TXT o ZIP (CSV, TSV o TXT compresso).<br><br>Questa informazione può essere utilizzata per aiutare i potenziali clienti a comprendere in che modo Search, Social e Commerce possono migliorare le prestazioni e risolvere le carenze evidenziate dall’analisi. Il team dell’account Adobe può inoltre utilizzare le informazioni approfondite per valutare gli account prima di procedere formalmente all’onboarding di un cliente. |
| [!UICONTROL Impression Share Lost] | ([!DNL PowerPoint] formato di presentazione) Indica quando il budget di un portfolio ha limitato la quota di impression per [!DNL Google Ads] e raccomanda di modificare il bilancio e la campagna. &quot;[!UICONTROL Multiple]&quot;. |
| [!UICONTROL Location Target Performance] | (Per [!DNL Google Ads] campagne; [!DNL Excel] formato del foglio di calcolo) La regolazione dell’offerta, i clic, il costo e [ricavi ponderati](/help/search-social-commerce/glossary.md#w-x) per ogni posizione target in un singolo portfolio. I dati sono disponibili per qualsiasi intervallo di date negli ultimi 90 giorni e puoi visualizzare i risultati giornalieri o un riepilogo per ogni campagna. |
| [!UICONTROL Match Type] | Grafici a schermo dei costi e [ricavi ponderati](/help/search-social-commerce/glossary.md#w-x) tra tipi di corrispondenza in un singolo portfolio fino agli ultimi 90 giorni. |
| [!UICONTROL Mobile Optimization] | ([!DNL PowerPoint] formato di presentazione) Mostra le prestazioni di un singolo portfolio per tipo di dispositivo negli ultimi 30 giorni, valuta le impostazioni correnti per le funzioni di ottimizzazione specifiche per dispositivi mobili e consiglia di regolare le impostazioni. |
| [!UICONTROL Model Accuracy] | Identifica il livello di previsione di spesa e ricavi per un singolo portfolio attivo o ottimizzato da parte di Search, Social e Commerce e identifica le origini delle imprecisioni dei modelli. La relazione include:<ul><li>A [!DNL PowerPoint] file con un riepilogo di costo previsto ed effettivo, clic, costo per clic (CPC), ricavi, ricavi per clic (RPC) e ritorno sull’investimento (ROI) nei sette giorni precedenti; dati aggregati su unità di offerta sovrautilizzate e sottoutilizzate nel mese precedente; grafici di tendenza che mostrano la precisione del modello di costo, la precisione del modello di ricavo e il costo previsto, clic, CPC, ricavi, RPC e ROI nel mese precedente rispetto ai valori effettivi; precisione del modello per livello di clic; unità di offerta sottoutilizzate o sovrautilizzate più elevate; consigli per migliorare la precisione del modello.</li><li>Un [!DNL Excel] file con la precisione del modello per ogni parola chiave e la precisione del modello per giorno.</li></ul>Questa informazione fornisce più dettagli rispetto ai report di accuratezza del modello a livello di portfolio, disponibili nella [!UICONTROL Portfolios] visualizzazione. |
| [!UICONTROL Normalized Sim (Combined)] | Restituisce l’allocazione di spesa ottimale per un numero specificato di livelli di spesa target (passaggi) per più portafogli. Questa informazione si basa su [ricavi ponderati](/help/search-social-commerce/glossary.md#w-x) in base alle simulazioni settimanali durante un determinato periodo di tempo. Puoi generare a) una singola simulazione per ciascun portfolio selezionato oppure b) una singola simulazione per più portfolio con lo stesso obiettivo e la stessa valuta. Le simulazioni vengono presentate in fogli di calcolo.<br><br>Per simulazioni singole e multi-portfolio, il foglio di calcolo include due fogli: 1) un grafico di andamento che mostra i ricavi ponderati normalizzati per ogni livello di spesa target (passaggio) e 2) una tabella che mostra una riga per ogni livello di spesa applicabile (passaggio). Per singole simulazioni di portfolio, il foglio di calcolo include 1) un foglio di riepilogo che elenca tutti i portafogli applicabili e il periodo di tempo e 2) due fogli per ciascun portfolio: a) un grafico di andamento che mostra i ricavi ponderati normalizzati per ciascun livello di spesa target (step) e b) una tabella che mostra una riga per ciascun livello di spesa applicabile (step). |
| [!UICONTROL Portfolio Launch] | ([!DNL PowerPoint] formato di presentazione) Identifica se un portfolio è pronto per essere lanciato (ha lo stato &quot;ottimizzato&quot;). Il rapporto include un confronto tra le prestazioni previste e le prestazioni effettive; la data di inizio del portfolio consigliata e un elenco di date che devono essere escluse dalle previsioni di costi e ricavi; il costo, i ricavi, il ricavo per costo e la conversione delle parole chiave nel tempo; informazioni su qualsiasi campagna la cui spesa è stata limitata nelle ultime due settimane; e un consiglio di avviare o meno il lancio. |
| [!UICONTROL Quality Score] | Mostra le tendenze del punteggio di qualità per le parole chiave con impression in [!DNL Google Ads] o [!DNL Microsoft® Advertising] negli ultimi 90 giorni. Include a) un [!DNL PowerPoint] file e b) [!DNL Excel] file che includono le 100 parole chiave con la spesa più elevata e (se applicabile) le 100 parole chiave con punteggi di qualità scadente.<br><br>**Note:**<ul><li>Quando il portfolio ha più di 20.000 unità di offerta, sono incluse solo le parole chiave con più di un clic negli ultimi 30 giorni (anziché 90 giorni).</li><li>La generazione di approfondimenti con dati per molte parole chiave richiede più tempo.</li></ul> |
| [!UICONTROL Query Cross Matching] | ([!DNL Google Ads] account) Identifica le istanze delle query di ricerca che [!DNL Google Ads] corrisponde a più parole chiave e fornisce suggerimenti su dove indirizzare il traffico. Utilizza i suggerimenti per migliorare le prestazioni delle parole chiave e, di conseguenza, migliorare la precisione del modello.<br><br>Le informazioni includono un foglio di calcolo delle query con corrispondenza incrociata, parole chiave corrispondenti e dati sulle prestazioni, diagrammi di rete per visualizzare le relazioni tra query con corrispondenza incrociata e parole chiave, il report sottostante delle query di ricerca e informazioni su come interpretare i dati. |
| [!UICONTROL Settings Audit] | Indica se i portfolio sono configurati in base alle best practice. Include:<ul><li>A [!DNL PowerPoint] file che mostra se le impostazioni chiave per tutti i portfolio e le campagne dei relativi componenti sono configurate in base alle best practice, se alcune campagne non sono assegnate ai portfolio e le conseguenze di deviazioni dalle best practice.</li><li>Un [!DNL Excel] file con una scheda separata per ciascuna impostazione di portfolio e impostazione di campagna chiave. I dati includono le impostazioni del portfolio per ciascuno dei portfolio e le impostazioni della campagna per ogni campagna che non soddisfa le best practice.</li></ul> |
| [!UICONTROL Time-of-day Analysis] | (Applicabile ai portafogli con [!DNL Google Ads] solo campagne di ricerca, visualizzazione o acquisto) Suggerimenti [!DNL Google Ads] modificatori di offerte a livello di campagna per diversi orari del giorno (modificatori di offerte di pianificazione degli annunci) in base alle prestazioni orarie per un singolo portfolio.<br><br>La descrizione include [!DNL Excel] cartella di lavoro con singoli fogli per [!UICONTROL Detailed Performance] (prestazioni per intervallo di tempo per campagna) e [!UICONTROL Recommended TBAs] (pianificazione degli annunci consigliata per ogni campagna). Puoi esportare la [!UICONTROL Recommended TBAs] Foglio direttamente al [!DNL Google Ads] editor. L’insight include anche una [!UICONTROL Powerpoint] file con dati e spiegazioni di supporto. |
